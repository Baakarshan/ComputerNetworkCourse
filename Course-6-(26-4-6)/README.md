## 3. 实现说明

### 3.1 题37：网桥自学习

输入帧字段：

- 源地址（4 bit）
- 源端口（1 或 2）
- 目的地址（4 bit，`0xF` 为广播）

处理逻辑：

- 先学习源地址到源端口映射（写入 MAC 表）。
- 若目的地址是广播：转发到另一端口。
- 若目的地址已在 MAC 表中：
  - 若目的端口与源端口相同：丢弃（过滤）。
  - 否则：定向转发到对应端口。
- 若目的地址未知：泛洪（两端口场景下等价为发往另一端口）。

程序输出每帧的目的端口决策，并打印实时 MAC 地址表。

### 3.2 题38：AP 部署模拟

采用“快速、简单”实现，重点满足题意输出：

- 建筑模型：`100m x 80m x 3层`，层高 `3.5m`。
- 墙体衰减（简化）：承重墙 `12dB`，普通隔断墙 `6dB`。
- 目标阈值：`>= -65dBm`。
- AP 选址：每层沿走廊中心线候选点穷举，选 2 个 AP（1 个 2.4GHz + 1 个 5GHz），目标“覆盖率高且重叠小”。
- 信道规划：
  - 2.4GHz 在 `1/6/11` 中穷举最小干扰组合。
  - 5GHz 在 `36/44/149` 中穷举最小干扰组合。
- 热力图：按楼层输出字符图（`#` 强、`+` 中、`.` 弱、空格无覆盖）。
